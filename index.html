어르신이 손글씨로 작성한 재능 신청서를 OCR·AI로 구조화해 QR 명함으로 만들고, 노(老)-노(老) 이웃을 안전하게 연결하는 한·일 초고령사회 돌봄 매칭 플랫폼

👥 5인 팀 프로젝트
Node.js MySQL Express REST API
🐙 GitHub ▶ 시연영상

프로젝트 개요

과학기술정보통신부 주최 '2026 글로벌 피우다프로젝트(한·일 노인돌봄 ICT 솔루션)'에서 개발한, 어르신끼리 재능을 나누는 노(老)-노(老) 연결 서비스입니다. 어르신이 종이에 손글씨로 적은 재능 신청서를 OCR·AI로 구조화해 QR 명함으로 만들고, 이웃이 QR을 스캔하면 개인 연락처 노출 없이 안심 매칭으로 이어집니다. 저는 백엔드에서 이 서비스의 핵심인 '안심 매칭 시스템'을 맡아, 어르신 간 신뢰를 기반으로 한 매칭 알고리즘과 관련 API 전반을 설계·구현했습니다. 예선 서면 평가를 통과해 본선에 진출한 프로젝트입니다.

핵심 기능 & 로직

  • ✦

    다중 요소 가중치 매칭 알고리즘 — 단순 거리 기반 매칭을 넘어, 주변 사용자와의 거리·평점·이용 후기 등 여러 요소에 가중치를 부여하고 조정해 실제 상황에 맞는 신뢰 기반 매칭 결과 도출

  • ✦

    MVC 구조의 매칭 모듈 설계 — matchModel / matchService / matchController / matchRoutes로 계층을 분리해 매칭 기능 전반을 유지보수 가능한 구조로 구현

  • ✦

    백엔드 정합성 고려한 API 설계 — 매칭 API가 회원·명함 등 다른 기능과 병목 없이 연동되도록 데이터 흐름과 백엔드 구조 간 정합성 확보

  • ✦

    확장 고려한 데이터 구조 — 매칭 이후 실제 만남까지 이어지는 스케줄링 로직으로 자연스럽게 넘어갈 수 있도록 데이터 구조 설계

이슈

거리만 가까우면 매칭 순위가 높아져, 신뢰도 낮은 상대가 먼저 노출됨

해결 → 평점·후기 데이터를 함께 가중치로 반영해 우선순위를 재조정, 신뢰할 수 있는 이웃이 먼저 연결되도록 로직 개선

이슈

어르신 간 매칭이라는 특성상 민감 정보(연락처·QR 식별자)가 노출될 위험

해결 → 후기 평점 기반 노출 우선순위는 살리되, 개인 식별 정보는 화면에 드러나지 않도록 매칭 흐름을 제안-수락-완료 버튼식으로 설계

사용 기술 스택

👥 4인 팀 프로젝트
Node.js Express ORS(OpenRouteService) Leaflet React Web Speech API
🐙 GitHub ▶ 시연영상

프로젝트 개요

거동이 불편한 어르신과 교통약자를 위한 배리어프리 길찾기 서비스입니다. ORS(OpenRouteService) API로 최단 경로와 계단 회피 경로 두 가지를 받아온 뒤, 좌표별 고도값으로 구간마다 경사도(%)를 직접 계산하고 계단 여부와 결합해 보행 비용을 산출합니다. 이 비용을 비교해 더 안전한 경로를 추천하며, 저는 프론트엔드(React + Vite)와 백엔드(Node.js + Express) 개발을 모두 담당했습니다.

핵심 기능 & 로직

  • ✦

    경사도 계산 엔진 — ORS가 반환한 좌표별 고도(elevation) 데이터로 구간별 경사도(%)를 고도차 ÷ 구간거리로 서버에서 직접 산출

  • ✦

    보행 비용 채점 & 경로 비교 — ORS에 최단 경로·계단 회피 경로를 각각 요청한 뒤, 경사도·계단 여부에 가중치를 곱해 보행 비용을 매기고 더 안전한 경로를 추천

  • ✦

    음성 길안내 — Web Speech API 기반 TTS로 경로 안내를 음성으로 제공해 접근성 강화

핵심 기능 & 로직

HTML5 CSS3 JavaScript Git
🐙 GitHub ▶ 시연영상

프로젝트 개요

지금 보고 계신 이 포트폴리오 웹사이트 자체가 하나의 프로젝트입니다. React나 Vue 같은 프레임워크 없이 순수 HTML/CSS/JS만으로 웜톤 컬러 시스템, 아코디언, 확장형 카드 등의 인터랙션을 직접 설계하고 구현했습니다. 컴포넌트 없이도 유지보수 가능한 구조를 만들기 위해 BEM 네이밍 규칙과 CSS 변수 기반 디자인 시스템을 일관되게 적용했습니다.

핵심 기능 & 로직

  • ✦

    CSS 변수 기반 디자인 시스템 — --coral, --honey 등 변수 하나로 전체 톤앤매너를 일괄 관리해 확장·수정 용이성 확보

  • ✦

    확장형(Expanding) 프로젝트 카드 — 팝업 없이 카드 자체가 그 자리에서 펼쳐지는 max-height/opacity 트랜지션과, 한 번에 하나만 열리도록 제어하는 아코디언 그룹 로직 구현

  • ✦

    반응형 그리드 레이아웃 — Hero/About/Projects 각 섹션을 CSS Grid와 clamp()로 구성해 데스크톱부터 모바일까지 최소한의 미디어쿼리로 대응

  • ✦

    부드러운 스크롤 내비게이션 — 고정 헤더 높이를 보정한 커스텀 스크롤 로직으로 앵커 이동 시 콘텐츠 잘림 방지

이슈

한 카드 안에 카드 전체 확장과 Troubleshooting 아코디언, 두 종류의 열고 닫힘 상태가 공존하며 로직이 얽힘

해결 → 카드 확장 여부와 내부 아코디언 여부를 각각 독립된 aria-expanded 상태로 분리해 관리, 상호 간섭 없이 동작하도록 구현
